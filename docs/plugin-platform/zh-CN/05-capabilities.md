# 05 — 能力与许可 (Capabilities & Consent)

OpenDray 遵循类似于 Chrome 扩展或 Android 的模式：插件在 `manifest.permissions.*` 中预先声明所需能力，用户在安装时会看到许可界面，并且每次桥接调用都会在运行时进行检查。

## 1. 能力分类 (Capability taxonomy)

| 键名 | 授予权限 | 默认值 | 许可描述 |
|-----|--------|---------|-------------------|
| `fs.read`  | 读取符合模式的文件 | 无 | "读取以下位置的文件：/<模式>/" |
| `fs.write` | 写入符合模式的文件 | 无 | "在以下位置创建/修改文件：/<模式>/" |
| `exec`     | 运行进程（通配符或全部） | 无 | "运行命令：<通配符>" 或 "在此机器上运行任何命令" |
| `http`     | 出站 HTTP（URL 模式或全部） | 无 | "向以下主机发起网络请求：<主机>" |
| `session`  | 读取或写入会话 | 无 | "读取终端输出" / "向终端发送输入" |
| `storage`  | 插件专用的键值对存储（隔离） | 无 | "存储插件特定数据" |
| `secret`   | 插件专用的机密存储 | 无 | "为此插件存储加密的凭据" |
| `clipboard`| 读取 / 写入设备剪贴板 | 无 | "读取剪贴板" / "写入剪贴板" |
| `telegram` | 通过配置好的桥接发送消息 | 无 | "通过您的 Telegram 机器人发送消息" |
| `git`      | 读取或写入工作区中的 git 仓库 | 无 | "读取工作区中的 git 历史" / "提交更改" |
| `llm`      | 使用配置好的 LLM 提供商（消耗令牌） | 无 | "调用您的 AI 模型（可能产生费用）" |
| `events`   | 订阅宿主事件（模式列表） | 无 | "监听以下事件：<模式列表>" |

## 2. 模式语言 (Pattern language)

### 路径模式 (用于 `fs.read`, `fs.write`)
- 基础变量：`${workspace}`, `${home}`, `${dataDir}`, `${tmp}`。
- 通配符语法：双星号 (`**`)、`*`、`?`、字符集 `[abc]`。
- 示例：`["${workspace}/**/*.md", "${home}/.config/myext/**"]`。
- **禁止：** 单独的 `/`、`/etc/**`、`/var/**`、`/proc/**`、`/sys/**`。安装程序会拒绝包含这些路径的清单。

### URL 模式 (用于 `http`)
- 方案必须是 `https://`，除非主机是 `localhost` 或明确的局域网 (LAN)。
- 主机通配符：`https://*.example.com/*`。
- 示例：`["https://api.github.com/*", "https://objects.githubusercontent.com/*"]`。
- 即使模式匹配，也会拒绝 RFC1918 / 链路本地地址，除非插件是从 `local:` 来源安装的（开发模式）。

### 命令模式 (用于 `exec`)
- 以空格分隔的 `命令 参数...` 通配符：`"git *"`、`"npm run *"`、`"cargo test --no-run *"`。
- 单个 `true` 将授予所有权限，并强制许可界面显示“运行任何操作”的红色横幅。

## 3. 安装时许可 UI

Flutter 外壳在安装时会渲染一个全屏面板：

```
┌─────────────────────────────────────────────┐
│  安装 "rust-analyzer-od"                    │
│  版本 v0.2.0，发布者：rust-lang             │
│  已验证发布者 · 签名有效                    │
│─────────────────────────────────────────────│
│ 此插件想要：                                │
│                                             │
│ • 读取并修改您工作区中的文件                │
│ • 运行命令：cargo *, rustc *                │
│ • 存储插件特定数据                          │
│                                             │
│ 危险操作：                                  │
│ • (无)                                      │
│                                             │
│ [ 取消 ]               [ 安装 ]             │
└─────────────────────────────────────────────┘
```

规则：
- 危险权限组包括：`exec: true`、匹配 `${home}` 根目录的 `fs.write`、无主机限制的 `http`、`session: "write"`、`telegram: true`。
- 危险授权需要第二次点击确认，并显示文本“我了解风险”。
- 未签名插件 + 危险授权 + `store: "community"` → 额外显示“来自未验证发布者”的警告。

## 4. 运行时提示

大多数能力在安装时静态授予。存在两种运行时提示：

1. **`clipboard: "read"`** — 浏览器和 iOS 需要用户手势才能读取剪贴板；插件在会话中第一次调用 `clipboard.readText()` 时会触发提示。
2. **计费周期内的第一次 `llm.*` 调用** — 每 30 天显示一次，包含当前的令牌消耗情况。

其他能力在运行时**不会**提示；被拒绝的调用会立即返回 `EPERM`。

## 5. 权限撤销

在“设置 → 插件 → <名称> → 权限”中：
- 每个能力都有开关（关闭 ⇒ 撤销；打开 ⇒ 重新授予）。
- 撤销权限会写入一个 `plugin.permission.revoked` 事件；活跃的 Web 视图在下次调用时会收到 `EPERM` 失败；边车进程会收到 `permissions/update` JSON-RPC 通知，如果忽略通知超过 2 秒，进程将被重启。

## 6. 审计日志 (Audit log)

每次桥接调用都会写入一个环形缓冲区（每个插件 1万条条目），包含以下内容：
```
{ ts, plugin, ns, method, caps: ['fs.read'], result: 'ok|denied|error', durationMs, argsHash }
```
可在“设置 → 插件 → <名称> → 审计”中查看。在磁盘上保留 30 天，然后进行垃圾回收 (GC)。

## 7. 实现说明

- 负责执行的 Go 包：`plugin/bridge/capabilities.go` (新增)。
- 数据库表 (M1 迁移)：
  - `plugin_consents(plugin_name, granted_at, manifest_hash, perms_json)` — 将许可固定到精确的清单哈希值，意味着扩大能力的清单更改将重新触发许可流程。
  - `plugin_audit(id, ts, plugin_name, ns, method, result, meta)`。

## 8. 插件永远无法获得的内容

即使授予了所有能力，以下内容仍仅限宿主使用：

- 写入 `plugins/` 目录（插件包在安装后是不可变的；更新需通过安装流程）。
- 写入 `kernel/store` 中除 `plugin_kv` 表及其自身的 `plugin_secret` 行以外的其他表。
- 访问另一个插件的 `storage` 或 `secret`。
- 不通过 `opendray.exec` 启动进程（允许在内核层调用边车 `os.exec`，但受 `exec` 能力的策略门控 —— 宿主在 Linux 上通过边车管理器的 rlimit/seccomp 包装器拦截系统调用，在 macOS/iOS 上则依赖于策略和审计）。
- 覆盖另一个插件在清单中声明的贡献点。
- 读取另一个插件的审计日志。

> **已锁定：** 第 8 节中的列表是硬性保证。任何新的宿主子系统在发布前必须说明其位于此线的哪一侧。
