# 10 — 安全 (Security)

## 威胁模型 (Threat model)

### 资产
- 用户工作区中的源代码（高价值）。
- 机密信息（API 密钥、令牌、云端凭据），存储在 `secret` 存储和 `EnvOverrides` 中。
- LLM 提供商计费（令牌消耗的是真实货币）。
- Telegram 机器人令牌和白名单聊天 ID。
- 宿主机（OpenDray 以用户权限运行）。

### 攻击者
1. **恶意的插件作者** — 编写插件来窃取机密或外泄代码。
2. **受损的插件市场** — 攻击者获得了对注册表仓库或发布者产物 URL 的写入权限。
3. **中间人攻击 (MITM)** — 拦截插件下载或更新检查。
4. **被盗的认证令牌** — 攻击者拥有用户的持有者令牌（Bearer Token），直接与桥接 API 通信。
5. **物理访问攻击者** — 设备未锁定，自行安装插件。
6. **供应链攻击** — 宿主插件的依赖项（npm 包、系统库）被篡改。

### 超出范围
- 内核级提权（OpenDray 以用户身份运行；插件可以执行用户能执行的任何操作，但受到能力门控的限制）。
- 插件市场注册表的拒绝服务（我们可以接受停机；安装是可选的）。

## 缓解措施矩阵 (Mitigations matrix)

| 威胁 | 缓解措施 |
|--------|-----------|
| 恶意作者 | 能力声明 + 许可界面；最小权限默认值；运行时桥接门控；审计日志。 |
| 受损市场 (注册表) | 已签名的发布者密钥；可选的产物签名验证；撤销列表。 |
| 受损市场 (产物 URL) | 注册表文件中的 SHA256 固定；签名检查；镜像支持。 |
| 中间人攻击 (MITM) | 注册表和产物仅限 HTTPS；官方宿主开启 HSTS；首次安装后对发布者密钥进行 TOFU 固定。 |
| 被盗持有者令牌 | 桥接 WebSocket 绑定到 Origin + Host 标头；令牌基于会话并在登录时重新签发；速率限制。 |
| 物理访问 | 移动端应用锁（PIN/生物识别）；首次使用敏感能力时进行提示。 |
| 供应链 | 插件包是自包含的；宿主插件无法在运行时获取代码（Web 视图受 CSP 限制，边车进程除非授予 `http` 能力，否则默认无网络访问）。 |

## 硬性保证 (插件永远无法做到的事情)

1. 写入 `plugins/` 安装目录。
2. 读取或写入另一个插件的 `storage` 或 `secret` 命名空间。
3. 覆盖另一个插件的贡献点。
4. 在用户未安装宿主形式插件的情况下逃逸 Web 视图沙箱。
5. 在没有 `exec` 能力或不匹配通配符的情况下启动进程。
6. 在没有 `http` 能力或不匹配 URL 模式的情况下发起出站网络调用。
7. 读取其声明的 `fs.read` 通配符之外的文件。
8. 写入其声明的 `fs.write` 通配符之外的文件。
9. 在 iOS/Android 上读取另一个应用的隐私数据（受操作系统沙箱限制）。
10. 通过直接调用宿主 HTTP 端点来绕过能力门控 —— 每个 `/api/plugins/*` 端点都通过激活时签发的桥接令牌将调用者绑定到特定插件。

> **已锁定：** 上述列表是硬性边界。任何新的桥接方法在发布前都必须经过能力审查，且不得违反这些保证。

## iOS App Store 策略

### 为什么这不属于 §2.5.2 规定的动态代码

- 声明式插件**不包含可执行代码** —— 仅包含原生 Flutter 外壳渲染的 JSON 数据。等同于主题文件。
- Web 视图插件在 `WKWebView` 中运行。Apple 明确允许在 Web 视图中加载 Web 内容。插件的 JS 是内容，而不是另一种运行时环境。这与所有 CMS、文档查看器和 Markdown 应用采用的模式相同。
- 宿主插件在 **iOS 上被禁用** (§06)。不会启动第二个进程。iOS 构建版本根本不包含管理器（Supervisor）。

### 额外的 iOS 特定约束 (在 iOS 构建中强制执行)

- `exec` 能力在 iOS 上不可用（无 `Process`/`posix_spawn` 接口）。桥接方法返回 `EUNAVAIL`。
- 插件包在 iOS 上不能包含二进制文件；安装程序拒绝 `form: "host"` 的插件包。
- Web 视图插件的下载内容存储在应用的 Documents 目录中，作用域限定在应用内；Apple 审核人员可以看到这些文件。
- 插件无法禁用操作系统的系统剪贴板提示。

### 提交审核时应包含的说明

- “插件是 UI 和数据扩展。所有可执行的插件代码都是 WKWebView 中的 Web 内容。应用默认自带一个空的插件目录；用户可以浏览显示描述的注册表，并安装预先批准的 Web 内容，类似于无头 CMS 或主题商店。在任何时候都不会加载动态原生代码。”

> **已锁定 (2026-04-19)：** 目前所有 11 个核心面板 (`plugins/panels/*`) 在所有平台上都内置在构建版本中，并由哈希值固定。发布后，根据遥测数据筛选出 5-7 个最常用的面板；其余的将移至仅限市场下载。这保证了 OpenDray 在 iOS 上无需安装任何内容即可完全使用，且符合审核安全要求。

## 用户信任等级与体验

| 等级 | 许可界面的处理方式 |
|-------|-------------------------|
| official (官方) | 无警告；绿色徽章。 |
| verified (已验证) | “已验证发布者”蓝色徽章；普通许可提示。 |
| community (社区) | “由 <发布者> 发布”文本；危险能力显示琥珀色/黄色横幅。 |
| sideloaded (侧加载) | “来自本地文件或 URL”红色横幅；对于任何危险能力，强制用户输入插件名称以确认。 |

## 机密处理 (Secrets handling)

- 插件机密使用 AES-GCM 加密存储在宿主的凭据存储密钥下。
- 机密值绝不会被记录到日志中，绝不会包含在审计日志的有效载荷中（仅记录键名），绝不会返回给 Web 视图的 Chrome DOM 检查器钩子。
- 宿主会从日志输出中遮盖任何与插件机密值匹配的字符串。
- 插件通过 `envVar` configSchema 条目设置的环境变量仅限定在启动的会话作用域内。

## 网络策略

- 桥接 WebSocket 接受来自以下位置的连接：
  - `app://opendray`（移动端 Flutter 应用的自定义方案）。
  - `https://<配置的前端主机名>`。
  - 仅当绑定到 localhost 时接受 `http://localhost:<端口>`。
- CSRF：每个变更端点都要求在 `Authorization` 中提供持有者令牌；不接受 Cookie。
- CORS：由前端主机配置驱动的允许列表。

## 审计与事件响应

- 中央日志文件位于 `plugins/.audit/<日期>.jsonl`。
- 通过“设置 → 插件 → 紧急按钮”以及 `OPENDRAY_PLUGINS_DISABLED=1` 环境变量提供紧急“禁用所有插件”标志。宿主启动并加载清单，但拒绝所有激活请求。
- 当检测到 `action: "uninstall"` 的撤销条目时，宿主执行卸载并发出包含所述原因的桌面/移动端通知。

## 不可扩展的内容 (硬性保证)

为了向插件作者明确说明，以免在设计时产生误解，以下内容不可扩展：

- 认证/登录流程。
- 会话启动代码路径（仅限贡献 `agentProviders` —— 宿主仍是启动者）。
- 桥接协议本身（插件无法提供传输层中间件）。
- 主题标记集（主题只能选择值，不能增加标记）。
- 能力系统（任何插件都不能削弱或提升另一个插件的能力）。
- 市场索引模式（插件可以镜像索引，但不能重新解释）。
- Web 视图 / 边车进程的操作系统级沙箱。

> **已锁定：** 这些是我们永远不想承担扩展成本的部分。在 post-v1 阶段，我们可能会开放其中一些（例如自定义认证提供者），但始终会放在新的能力之后。
