# OpenDray Mobile

Capacitor-wrapped React SPA targeting iOS and Android (phone + tablet).

Architecture is documented in [`docs/adr/0015-mobile-platform.md`](../../docs/adr/0015-mobile-platform.md).

## First-time setup (after a fresh clone)

The web bundle is in this repo, but the **native iOS and Android projects** (`ios/` and `android/`) are not committed — they're generated by Capacitor on first checkout. You only need to do this once per developer machine.

### Prerequisites

| Need | For | Install |
|---|---|---|
| Node 22 + pnpm 10.27 | Web bundle build | nvm / corepack |
| Xcode 15+ + Command Line Tools | iOS build | App Store |
| CocoaPods 1.15+ | iOS dependencies | `sudo gem install cocoapods` |
| Java 17 + Android SDK 34 | Android build | Android Studio |

### Generate native shells

```bash
# From the repo root
pnpm install                           # links workspace packages
pnpm --filter mobile build             # produces dist/

# Generate ios/
pnpm --filter mobile exec cap add ios

# Generate android/
pnpm --filter mobile exec cap add android
```

Both commands are idempotent — safe to re-run if a shell needs rescaffolding.

After this initial setup, `ios/` and `android/` exist locally. The artefacts that change with every build (Pods, .gradle caches, derived data, keystores) are gitignored; the project files (Xcode project, Gradle config, AndroidManifest, etc.) live alongside the workspace.

### Sanity-check the dev build

```bash
# iOS simulator
pnpm --filter mobile build && pnpm --filter mobile exec cap sync ios
pnpm --filter mobile exec cap open ios   # opens Xcode
# In Xcode: select a simulator, hit Run

# Android emulator
pnpm --filter mobile build && pnpm --filter mobile exec cap sync android
pnpm --filter mobile exec cap open android   # opens Android Studio
# In Android Studio: select an AVD, hit Run
```

The current placeholder screen (B1) confirms three things:

1. Capacitor's Vite + WebView pipeline is wired
2. `shared-ui` primitives render correctly under the mobile entry
3. Tailwind v4 + design tokens are applied

If you see "OpenDray / Mobile shell — phase B1" with the platform name and a working Button, B1 is correctly installed.

## Day-to-day commands

```bash
pnpm --filter mobile dev               # Vite dev server on :5174 (browser preview, no native shell)
pnpm --filter mobile build             # Production-quality web bundle into dist/
pnpm --filter mobile exec cap sync     # Copy dist/ into ios/ and android/ (run after each build)
pnpm --filter mobile exec cap run ios -l --external   # Live-reload to a connected iOS device
pnpm --filter mobile exec cap run android -l --external
```

The `dev` script is for fast iteration in a desktop browser. For real native testing you must `build` then `cap sync` then run from Xcode / Android Studio.
